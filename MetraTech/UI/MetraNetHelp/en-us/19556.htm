Example of a Multipoint Target

In this scenario, the rating data is submitted for MetraFlow execution using several files where there is a parent-child relationship between the data in the files. This example is for an audioconference multipoint service, in which one data file contains usage for conferences as a whole (the parent), and another data file contains usage for the conference legs.

Suppose that the target for the nested files is configured as:

Format

Order

Filter

Conditional

--arg AUDIOCONF="$(FILE)"

1

audioConf

ALWAYS

--arg AUDIOLEG="$(FILE)"

2

audioLeg

ALWAYS

C:/fls/ MetraFlowScripts/test_meter_multi.mfs

3

 

ALWAYS

Before the script can run, both files must be present. FLS will wait till the second file has been moved into the new folder:

ctrl0001.multimeter.audioLeg.txt

ctrl0001.multimeter.audioConf.txt

There is a nonconfigurable 5-minute timeout on nested files being moved to the new folder. If files have old modified dates, FLS will fail the job if all folders are not present.
