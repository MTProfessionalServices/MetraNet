FLS Execution Sequence

All FLS data processing scenarios share the same basic sequence:

  1. Select one MetraNet server to run the File Landing Service.
  2. Use MetraControl to specify the incoming directory and the executable targets and arguments.
  3. Start the FLS Service.
  4. Copy or create a file using the file naming convention and containing data in the FLS new folder.
  5. The job becomes active, and FLS identifies it by a control number. The file moves to the inprogress folder.
  6. FLS begins processing the data contained in the file using the target executable.
  7. MetraControl shows the status of the job.
  8. When the job is finished, FLS moved to the done folder.
