If you selected "I will use SQL directly to identify the records to be backed out", this screen provides instructions on specifying which transactions to back out using direct SQL against the database.

Only click Next Step: Identify & Analyze if you have finished specifying with SQL Server Query Analyzer. Otherwise, your backout will have no transactions.

If you make a mistake and SQL identifies the wrong transactions, there are no consequences at this point in the backout.

Click Abandon This Backout And Start Over in the next screen. Your backout has no effect on MetraNet's main database tables.
