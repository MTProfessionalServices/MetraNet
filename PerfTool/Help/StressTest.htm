Stress Test

Overview

The Stress Test simulates a large volume of varied tests that may happen concurrently. Each test type has average calls per hour. We use a random Poisson distribution function to determine the start of the next individual test. This means that tests aren't called precisely every N time-units.

The Stress Tester is currently coded to use 4 background worker threads to run individual tests.

Configuration

The Stress Test is configured under the preferences menu. Each test can be enabled or disabled. The text box per test is the number of calls per hour on average.
