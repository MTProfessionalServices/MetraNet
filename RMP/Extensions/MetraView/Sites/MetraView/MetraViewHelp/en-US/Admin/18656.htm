Bill Settings

The Bill Settings component has properties that determine the billing interface for the whole site.

Parameter

Default Value

Description

Allow Online Payment

True

Permits subscribers to make payments online.

Allow Saved Reports

True

Permits subscribers to save their own filters as a report.

Allow Self-Care

True

Permits the user to update their contact and account information.

Inline Adjustments

False

Shows adjustments inline or as separate line items in the bill.

Usage Note
This setting applies to prebill adjustments only. Those are adjustments to the bill's subtotal before tax. The default is to shows these adjustments on a separate line labeled Prebill Adjustments. If you set Inline Adjustments to be True, it means the adjustment charges or credits are not broken out as a separate amount, but are rolled into the subtotal. More Details

Inline Tax

False

Shows taxation inline or as a separate line item in the bill.

Usage Note
The default is to shows taxes on a separate line labeled Tax. If Inline Adjustments is False also, a line item also appears for Tax Adjustments. If you set Inline Tax to be True, it means that taxes are not broken out as a separate amount, but are rolled into the subtotal. More Details

Show Only Hard-Closed Intervals

False

Shows estimated billing data if a billing interval is still open. By setting this to true, subscribers can see the open interval usage on the main Bill & Payments page (for both by-product and by-folder views) and on the History page.

Warning
This data may be inaccurate due to unprocessed charges. Depending on how your billing operations are managed, this could delay the closing of disputes and cause user dissatisfaction.

MetraView currently supports a single instance of this component.
