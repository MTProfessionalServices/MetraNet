Adding, Editing, or Deleting Components

Site components are potential interface items until you add them and configure them with defining properties. When you add a component, you are creating an instance of that component. After it's added, you edit the component to specify a particular set of attributes: such as page location, column position, size, color, and so on. You may need to add multiple component instances to one page or several pages in your web application. For example, a particular widget, such as calendar, might be useful on several pages.

To define or view a component:

  1. From the Site list, click View Relationships.

    This opens the Relationships for <YourSite> page - a grid with tabs for each component you add, view, or modify in your custom site. (The site has a one-to-many relationship with each component, so you can potentially add multiple instances of each component.)

    The tabs are: Bill Setting, Dashboard, Entry Point, Product View Mapping, and Report Inventory.

  2. For each component type, select Add, Edit, or Remove.

    For example, to add a new dashboard instance:

    1. Click the Dashboard tab and then click Add.

      A new window opens the component list.

    2. Click Add again to create the new component instance.
    3. Proceed to edit the new component.

Deleting or removing a component can have serious consequences. Never do this in a production environment.

Creating new site instances is not supported in this release.
