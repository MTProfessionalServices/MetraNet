Metering and Batch IDs

A script that reads usage from a file and places the data in tables (t_session, t_svc_..., t_message, and t_session_set). Rating of the usage does not take place in MetraFlow. The rating and batch record creation occurs when the pipeline picks up the usage data and rates it.

readTravel: import[

filename=$inputFile,

format="myRec (

c_account text_delimited_nvarchar(delimiter=','),

c_duration text_delimited_base10_decimal(delimiter=','),

c_language text_delimited_enum(enum_space='travelspot.com/TravSpotEnums', enum_type='Languages', delimiter=','),

c_ConferenceID text_delimited_base10_int32(delimiter=','),

c_ConferenceName text_delimited_nvarchar(delimiter=crlf))",

mode="sequential"];

meterTravel: meter[service="travelspot.com/TranslationService",

collectionIDEncoded=$batchID generateSummaryTable=false];

In the above script, the meter data is associated with a batch, but MetraFlow does not create the batch record. The batch record is created when the pipeline picks up the data. For example, if the pipeline was down and you ran the above example, and then used MetraControl/Batch Management, you would not see the batch. If you then started the pipeline, the batch would be created. The "expected" field of the batch, telling the number of records in the batch of usage, would be zero.
