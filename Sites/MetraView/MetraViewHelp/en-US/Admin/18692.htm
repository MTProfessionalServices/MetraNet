Adding Site Instances

MetraNet installs the default MetraView site. Using MetraView Admin, you can customize this site or create new instances of MetraView sites. Each site has its own branch of folders and files (under RMP\Extensions). This corresponds with a unique IIS virtual directory - and therefore URL. Each site can have different nested components and other customizations. A single Admin interface lets you manage all the site instances you create and their nested components.

Creating new site instances is not supported in this release.

Create the Site in ICE

The first stage in creating a new site is to use the ICE site creation tool.

  1. From ConfigExplorer click the Sites panel.
  2. Right-click Sites » Add.
  3. In the Add Site dialog:
    • Enter the name of the site. This determines the virtual directory name also.
    • Select MetraView as the extension and the existing site to copy.

    When you click Add ICE creates:

    • A new folder tree for your site under RMP\Extensions\MetraView\Sites.
    • It also creates a new virtual directory for your site in IIS under the Default Site.
  4. Now you have added the site, there are a number of manual configuration steps that need to be done.

    Please ask your MetraTech Professional Services consultant for details.

Edit the Site Instance in MetraView

Creating new site instances with the MetraView Admin user interface is not supported in this release. You should create the site in ICE first. (After this, you can edit the new site with MetraView.)

After you have added the site with ICE and completed the manual configuration steps, you are ready to create a new site instance in the Admin section (of the original MetraView site):

  1. Create the Site in ICE.
  2. Edit the new site properties.
    1. Log into MetraView as MetraViewAdmin.
    2. Select the Admin tab and click Edit for your new site.

  3. Edit the Site Components.

    Expand the site to view its details and click the components you want to edit.
