Adding Site Instances

MetraNet installs the default MetraView site. Using MetraView Admin, you can customize this site or create new instances of MetraView sites. Each site has its own branch of folders and files (under RMP\Extensions). This corresponds with a unique IIS virtual directory - and therefore URL. Each site can have different nested components and other customizations. A single Admin interface lets you manage all the site instances you create and their nested components.

How to Locate MetraView Site Files

The location of MetraView site files is RMP\Extensions\MetraView\Sites\

  • The location of the default site files is: RMP\Extensions\MetraView\Sites\MetraView
  • A new site instance creates a new branch: RMP\Extensions\MetraView\Sites\<NewSite>

For a detailed breakdown of files by subfolders, see MetraView Directory Structure.

Creating a Site Instance

To create a new site instance:

  1. Select Configure Site » Site component.
  2. Click Add under the Site section

  3. Edit the new Site properties.
