Debugging Page.WaitForDownLoad

When debugging the following code :

Page.Controls("butAddCustomer","Anchor").Click
Page.WaitForDownLoad "AddCustomer.asp"

You should use the F10 key twice in a row with no interruption, because if you execute the first line and wait, the page will be automatically downloaded and when you press F10 to execute the line Page.WaitForDownLoad() the method will time out because the page has already been downloaded.

if you see that the page has already been downloaded you can skip the method Page.WaitForDownLoad() by using the Visual Studio debugger command Set Next Statment Ctrl+Shift+F10.
