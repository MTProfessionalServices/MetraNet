Introduction

W3Runner automates testing to ensure that Web Applications running under Microsoft® Internet Explorer 5.0, 5.5, 6.0 work as expected. You record and edit the user interactions with the VBScript Language and afterwards you can run your test. To program the HTML user interface objects W3Runner supplies a simplified, Visual Basic Like programming interface through 2 COM Objects. Have a look at the programming sample .

Writing W3Runner VBScript Test File
In design mode you use W3Runner as a simplified Microsoft® Internet Explorer browser. The Watch Window allows you to view information about most of the objects in the current pages. If the record mode is on :

the VBScript source code to set and load the URL is automatically generated.
by pressing the ALT key and clicking on the object you generate the default VBScript source code that goes with the object.

Because you use VBScript :
You can write any kind of check point, data-driven test or regression tests. W3Runner supplies by default the class CDB which allow to access most of the SQL Server through ADO and the class CCSVFile which allow to read a CSV file. .

You can write any analysing process and produce report.

About VBScript

VBScript is open to COM objects. Since VBScript 5.1, the notions of dynamic evaluation (eval() function), class, instance, property and include are available. With the basics of object oriented programming, large tests script can be written, debugged and maintained.

W3Runner is compatible with Microsoft® Visual Studio Debugger or Microsoft® Script Debugger.

What are the Microsoft® Browser compatible with W3Runner ?
W3Runner supports Internet Explorer 5.0, 5.5 and 6.0. Some features are not available with Internet Explorer 5.0. See the FAQ page for more information.
