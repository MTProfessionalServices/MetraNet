Introduction

W3Runner automates testing to ensure that Web Applications running under Microsoft® Internet Explorer 5.0, 5.5, 6.0 perform as expected. You record and edit the user interactions with the VBScript Language, and afterwards run your test.

To program the HTML user interface objects W3Runner supplies a simplified, Visual Basic Like programming interface through 5 following COM objects : Page, Control, W3Runner, Report and HTTP.


Writing W3Runner VBScript Test File
In design mode you use W3Runner as a simplified Microsoft® Internet Explorer browser. Press the Record Icon and enter an URL. The Watch Window allows you to view information about most of the objects in the current page or frames.

  • Recording URL : Enter the URL in the address combo box and press Enter. The VBScript source code which sets and loads the URL is automatically generated.

  • Recording the User Actions : Press the ALT key and click on the object you want the VBScript source code to be generated. The generated VBScript source code appears in the Watch Window.

Because you use VBScript you can write any kind of check point, data-driven test or regression test. W3Runner comes by default with a set of VBScript classes called "Extension":

Database

The CDB class allows access to most of the Database through ADO.

CSV File

The CCSVFile class allows one to read and write CSV files.

Test Report

The CTestReport class allow one to generate an HTML or TEXT report about the current test script.

Web Services

The classes CMSSoapSDKTest and CHttpSoapTest allow to Test WebServices calls using the Microsoft SOAP Tool kit or direct HTTP call.

Programming

FIFO, Stack and Collection objects are available.

You can use and modify theses classes and build your own test library to test your Web Applications, Web Service and any kind of HTTP dialog.

W3Runner supports ActiveX and Macromedia ShowWave Flash Control (see Welcome Page).


About VBScript
Since VBScript 5.1, the notions of dynamic evaluation (eval() function), class, property and include are available. With the basics notion of object oriented programming, large tests script can be written, debugged and maintained.

Microsoft® Browser compatible with W3Runner.
W3Runner supports Internet Explorer 5.0, 5.5 and 6.0. Some features are not available with Internet Explorer 5.0. See the FAQ page for more information.

Microsoft® Windows® operating systems compatible with W3Runner.

  • Windows® 2000 operating system

  • Windows® NT4 operating system
  • Windows® 98 and Windows® ME operating systems are not supported, but we did not find any specific problems.

What about Microsoft® .NET ?
Web applications developed with ASP.NET can be tested with W3Runner.
