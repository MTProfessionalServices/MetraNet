AutoFireEvent

Public Property Get AutoFireEvent() As Boolean
Public Property Let AutoFireEvent(v As Boolean)

Description:
Get or set if W3Runner should fire some missing JavaScript events. We have noticed that the Microsoft® WebBrowser control (OCX) does not raise the following events when the following properties are set programmatically:

  • Combox, event OnChange() when the property Value is set.

  • Radio Button, event OnClick() when the property Value is set.
  • Check Box, event OnClick when the property Value is set.

To compensate this problem, if the property AutoFireEvent is TRUE then W3Runner raises the event. The default value of the property AutoFireEvent is TRUE.
This only matters if you are using in your application the JavaScript ComboBox OnChange event or JavaScript Radio Button or CheckBox OnClick event. It is possible to cancel this behavior and raise the event yourself, as seen in the sample below.

Remark:
Microsoft® Internet Explorer 5.0 does not support this method, therefore the missing JavaScript event cannot be fired.

Sample:

Page.AutoFireEvent = FALSE
Page.Controls("MyCheckBox","CheckBox").Value = vbChecked
Page.Controls("MyCheckBox","CheckBox").FireEvent "OnClick"
Page.AutoFireEvent = TRUE
