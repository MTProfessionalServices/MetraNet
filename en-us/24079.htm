Step 1: Obtain the Query Details

The first step when defining a new report is to obtain the name of production-ready query and the description of what that query does.

For example: __GET_ACCOUNT_CONTACT_SUMMARY_REPORT__

You should use ICE to view descriptions, the SQL code, and the parameters. See Query Management with ICE

These queries exist in RMP\Extensions\DataExport\Config\SQLCore\Queries\Custom. The description exists in the INFO file.

Important

All report queries should be tested in a development or staging environment before making them available to senior MetraControl operators.

For more information, Search Documentation Set: Setting Up Data Export Reports
